# First Steps

Congratulations on setting up your SaaS starter! This guide will walk you through the key features and help you understand how everything works together.

## Understanding the Architecture

Your SaaS starter is built with a modern, scalable architecture:

- **Frontend**: Next.js 14 with App Router and React Server Components
- **Backend**: Supabase for database, authentication, and APIs
- **Styling**: Tailwind CSS for responsive, utility-first styling
- **Content**: MDX for rich content and documentation
- **Type Safety**: TypeScript throughout the entire stack

### 4. Create Database Tables

Add custom tables for your application:

1. Go to your Supabase dashboard
2. Navigate to "Table Editor"
3. Create new tables with appropriate columns
4. Set up RLS policies for security

Example table creation:

```sql
-- Create a simple posts table
CREATE TABLE posts (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  title text NOT NULL,
  content text,
  user_id uuid REFERENCES auth.users(id),
  created_at timestamp DEFAULT now()
);

-- Enable RLS
ALTER TABLE posts ENABLE ROW LEVEL SECURITY;

-- Create policy
CREATE POLICY "Users can view their own posts" ON posts
  FOR SELECT USING (auth.uid() = user_id);
````

### Creating a Protected Route

```typescript
// app/admin/page.tsx
import { redirect } from "next/navigation";
import { createServerComponentClient } from "@supabase/auth-helpers-nextjs";
import { cookies } from "next/headers";

export default async function AdminPage() {
  const supabase = createServerComponentClient({ cookies });
  const {
    data: { session },
  } = await supabase.auth.getSession();

  if (!session) {
    redirect("/auth/sign-in");
  }

  return <div>Admin Content</div>;
}
```
